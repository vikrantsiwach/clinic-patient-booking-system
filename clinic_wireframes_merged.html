How the missed call → SMS reply flow works end-to-end. Zero cost to patient. No internet required.

Free for you. No internet. Any phone.

How to Book in 3 Steps
1
Give a Missed Call
Call the clinic's dedicated number and hang up. Costs you ₹0. The call won't connect — just ring and cut.
📞 0120-XXXXX-XX

Clinic's dedicated missed call number

2
Receive SMS with Slot Options
Within 30 seconds, you'll receive an SMS with 3 available appointment slots to choose from.
"Dr. Sharma Clinic: Reply to book — Reply 1A3B for Mon 16 Mar 10:20AM, 2A3B for Mon 16 Mar 11:00AM, 3A3B for Tue 17 Mar 9:00AM. Valid 15 mins."
3
Reply to Confirm
Reply with your choice code (e.g. 1A3B) and your appointment is instantly confirmed with a reference number.
⏱ Slot options expire in 15 minutes. If you don't reply, just give another missed call.
System Flow
1
Webhook TriggerMSG91 detects the missed call (0–3 second duration) and immediately fires a POST webhook to your backend API endpoint.
2
Spam Checks (6 layers)Backend runs all spam filters synchronously: blacklist → rate limit → number validation → velocity check → duplicate appointment check. Takes ~50ms.
3
Slot ComputationSystem fetches next 3 available slots using the same slot generation algorithm used by the online booking engine. No separate logic needed.
4
Session TokenA unique 4-char token (e.g. "A3B7") is generated and embedded in the SMS options. Patient's reply must include the token — prevents spoofing.
5
Auto-ExpiryA cron job runs every 5 minutes to expire sessions older than 15 minutes. Expired sessions release their tentatively held slots.
6
New Number DelayFirst-time callers (not in patients table) wait 60 seconds before receiving the SMS. Invisible to humans, blocks automated dialers.

Spam Protection Layers NEW

6 layers of protection. Each layer runs in sequence. A call blocked early costs nothing in SMS credits.

🛡️ Filter Pipeline (per incoming call)
Filter StatusLive
🌐

Layer 1 — Telephony Filter

PROVIDER
Rejects non-Indian numbers, international VoIP, and calls longer than 4 seconds. Handled by MSG91 before webhook fires. Zero cost even if triggered.
⚫

Layer 2 — Blacklist Check

DB LOOKUP
Checks phone_blacklist table. Permanent bans and temporary bans with expiry. Staff can add numbers via admin panel. Auto-populated after 5 violations.
⏱

Layer 3 — Rate Limit

2/day · 5/week
Max 2 missed calls from same number in 24hrs, 5 in 7 days. Silent block — no SMS, no error. Configurable in admin settings without code change.
📋

Layer 4 — Number Validation

INSTANT
Format check (must be 6/7/8/9 prefix, 10 digits). MSG91 DND registry check. Landlines rejected instantly. Takes <5ms.
⚡

Layer 5 — Velocity Detection

15 calls/10 min
System-wide burst: if >15 calls in any 10-minute window, auto-pause for 30 mins + SMS alert to admin. Catches SIM farm attacks rotating multiple numbers.
📅

Layer 6 — Duplicate Guard

BUSINESS LOGIC
If patient already has an upcoming booked/confirmed appointment, skip new booking and SMS their existing appointment details instead. Protects slot inventory.
📊 All blocked calls are logged in missed_call_events with block_reason for admin analytics.
What Gets Through?
✓
Genuine PatientPasses all 6 layers. Gets SMS within 30 seconds. Books appointment with one reply.
✗
International AutodialerBlocked at Layer 1 (telephony). Never reaches your server. Zero cost.
✗
Repeat Spammer (Same SIM)Blocked at Layer 3 after 2 calls/day. Auto-blacklisted after 5 violations.
✗
SIM Farm AttackMultiple numbers rotating quickly hit Layer 5 velocity detection. System pauses + admin alerted within 10 minutes.
→
Accidental Double CallPatient who already has an appointment gets their existing appointment details shown. No duplicate booking created.
→
Token SpoofingSomeone who randomly texts the clinic number without the token gets no response. Session token required to book.

📞 Missed Call System 6 NEW ENDPOINTS
POST /api/missed-call/webhook System/Webhook MSG91 missed call event
Description

Receives webhook from MSG91 when a missed call is detected. This is the entry point for the entire missed call flow. Secured via HMAC signature verification (not a JWT — MSG91 signs requests with a secret).

Request Body (from MSG91)
{ "mobile": "+919876543210", "callDuration": 2, "callId": "msg91-abc-123", "receivedAt": "2026-03-16T04:11:00Z" }
Processing Pipeline
  • Verify MSG91 HMAC signature — reject with 401 if invalid
  • Log event to missed_call_events immediately (even if blocked)
  • Run check_spam_filters() PostgreSQL function (blacklist → rate limit → validation → velocity → duplicate)
  • If blocked: update event status, return 200 OK silently (no error feedback to provider)
  • If first-time caller: schedule SMS send after 60-second delay
  • Fetch next 3 available slots using internal slot engine
  • Generate 4-char session token via generate_session_token()
  • Create missed_call_sessions record with slot options and token
  • Send options SMS via MSG91 API
  • Always return 200 OK — never 4xx/5xx (prevents MSG91 retry storms)
Response (always 200)
{ "received": true }
POST /api/missed-call/reply System/Webhook Inbound SMS reply from patient
Description

Receives webhook when patient replies to the options SMS. MSG91 fires this when an inbound SMS arrives on the clinic's virtual number.

Request Body (from MSG91)
{ "mobile": "+919876543210", "message": "1A3B", "receivedAt": "2026-03-16T04:13:00Z" }
Processing Logic
  • Parse reply — extract option digit (1/2/3) and token (e.g. "A3B")
  • Look up active pending session by (patient_phone + token) — return error SMS if not found
  • Check session.expires_at > NOW() — send expiry SMS if session expired
  • Check if CANCEL — cancel matching upcoming appointment, send confirmation
  • Validate chosen_option is 1, 2, or 3
  • Create appointment via same engine as POST /api/appointments (booking_channel = "missed_call")
  • If slot taken since session was created: send alternative-slots SMS
  • On success: update session status to 'completed', send booking confirmation SMS
  • Queue 48h and 2h reminder notifications
Confirmation SMS Sent to Patient
"✅ Confirmed! Dr. Sharma on Mon 16 Mar at 10:20 AM. Ref: APT-2026-01424. Address: Civil Lines, Meerut. Reply CANCEL to cancel."
GET /api/admin/missed-call/analytics Admin JWT Daily/weekly analytics
Query Params

Response 200

Request Body
{ "maxCallsPer24h": 2, "maxCallsPer7d": 5, "sessionExpiryMins": 15, "burstThreshold": 15, "systemPauseMins": 30, "newNumberDelaySecs": 60, "systemEnabled": true }

POST /api/admin/missed-call/blacklist Admin JWT Add number to blacklist
Request Body
{ "phone": "+919876543210", "reason": "manual_staff", "notes": "Repeated prank calls", "isPermanent": false, "expiresAt": "2026-04-16" // null if permanent }

  • Audit log entry created for every blacklist addition
